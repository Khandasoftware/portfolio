Backend

WordPress provide a well documented api to allow programmers to customize or add functionality of the CMS.

  • Custom Post Types

    You can simply create your own models by registering your own post types and then use template files in your theme to route and render your data to the frontend.

  • Custom Fields / ACF

    You can add you own custom fields using Post meta or the most commonly used custom fields plugin called ACF.

  • Query Builder / WP Query Class

    WordPress provides an intuitive class to do all kinds of model filtering, you can query data by post type, date,sort,order,aggrigate atc.

  • Hooks

    Actions and Filters are a way in which you can add functionality to WordPress without altering the core code of WordPress. Hooks in WordPress are typically associated with the Observer or Publish-Subscribe design pattern.

    Action hooks are used to perform actions at specific points in the WordPress execution process. Actions don't return values; they are used to trigger events. For example, you can use an action hook to run a function when a post is published. Filter hooks, on the other hand, are used to modify data before it is displayed or used.

  • API Calls

    You can use PHP in WordPress to make HTTP requests to external APIs using functions like wp_remote_get or wp_remote_post.

    Sometime you have to pull WordPress data into a website that does not use PHP like PrintWild, I had create a custom route to display specific blog posts per based on a category associated with the product in the main site.

  • SQL / WPDB

    I always try to avoid using $wpdb and only use it when data operations can't be achieved using built-in WordPress functions and APIs.

HTML

Another huge part of my job is website perfomance and HTML plays a huge part when it comes to accessibility, loading speed and SEO. I use Google Lighthouse and Google Search console to find out what may need to be improved. I also use SEMRUSH to anaylise perfomance of each website. I would write a report and then fix the issues.

Sometimes we understand how capable HTML5 is, We tend to think of it as a way to structure our data in the UI but there is so much that HTML can do such as:

  • Form Validation

    For client side validation, some HTML attributes can help you validate you user data before sending it to the server for further processing. Some of these attributes such as Required ( to ensure input id mandetory ), Min, Step and Max, multiple and more.

  • SEO Meta Data

    Huge part of my daily job is to run SEMRUSH and fix on page SEO related issues to make sure Web crawlers do not have hard time trying to index clients website pages. Cannonical tags, Alt image attributes, meta robot are all tags that I'd troubleshoot.

  • Semantic HTML

    I link to make use of semantic HTML ( instead of DIVs/Spans) to structure the data in a meaningful for accessibility and Web crawlers. Aria labels are great for accessibility too.

  • Reactive HTML

    You can get make use of accessible and lightweight accordion by using details tag.

  • Image Optimization

    I wrote a blog about Image Optimization and huge part of it was about how can leverage HTML lazyload,SRCSET and Picture tag.
